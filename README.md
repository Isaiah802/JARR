#### Description:
The JARR Containers website is a project I created to serve as an online platform for JARR Containers, a family-owned business based in La Vernia, TX. The website provides detailed information about the products and services offered, including used and one-trip containers, general moving services, and free shipping within a 50-mile radius.

### Key Features:
1. **Product Browsing**: 
   - Customers can view the inventory of 40ft and 20ft containers, including both used and one-trip options.
   - Interactive links provide detailed information about each container.

2. **Rustic Design**:
   - The website design embraces a rustic family vibe, reflecting the values and ethos of the business.

3. **User-Friendly Navigation**:
   - The homepage includes a clickable logo to return to the top from any page.
   - Links to specific pages for products, testimonials, FAQs, and contact information are readily accessible.

4. **Customer Engagement**:
   - A contact form allows users to submit inquiries directly. 
   #took out due to not having persistent storage on hosing platform. 
   - Testimonials and positive customer feedback are highlighted.
   - An FAQ section addresses common concerns, such as shipping policies and container conditions.

5. **CRUD  Integration**:
   - Create Read Update Delete
   - Admin login
   - Admin panel 

6. **Social Media Links**:
   - A direct link to the business's Facebook page fosters community interaction.
   - Currently, the company has no facebook. 

7. **Mobile Optimization**:
   - The website is fully responsive, ensuring a seamless experience across devices.

### Technologies Used:
- **Frontend**: HTML, CSS, JavaScript
- **Backend**: Python with Flask
- **Database**: SQLite 
- **Hosting**: Render ("Am considering Railway. Open to comments on this topic.")

### File Structure:
- **templates/**: Contains HTML templates for all the pages of the website.
- **static/**: Includes static assets such as CSS, JavaScript, and images.
- **app.py**: The main Flask application file, managing routing and server-side logic.
- **README.md**: This file, which documents the project.
- **requirements.txt**: Lists dependencies for the Python environment.

### Design Choices:
- The rustic theme was chosen to emphasize the family-owned nature of the business.
- Mobile-first design principles were applied to ensure accessibility for users on various devices.
- The use of Flask was motivated by its simplicity and flexibility for small-to-medium projects.

### Challenges and Learning:
Building the website required integrating various technologies and balancing aesthetics with functionality. I learned:
- How to create interactive and user-friendly web pages using Flask.
- The importance of responsive design for mobile optimization.
- Techniques for creating and maniging dynamic input from end users(admin).
